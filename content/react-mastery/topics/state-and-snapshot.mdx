## 한 장 핵심

React state는 컴포넌트 함수 안의 일반 변수처럼 즉시 바뀌는 값이 아니다. `setState`는 현재 render 안의 변수를 바꾸지 않고, React에게 다음 render를 요청한다. React가 컴포넌트를 다시 호출할 때 그 render에 맞는 새 state 스냅샷을 넘겨준다.

그래서 React에서 state를 읽는 코드는 항상 "이 render가 받은 값"을 읽는다. 이벤트 핸들러, timeout 콜백, promise 콜백도 자신이 만들어진 render의 props와 state를 닫아둔다. 이 모델을 이해하면 stale closure, 연속 setState, 객체 mutation 문제를 훨씬 정확하게 디버깅할 수 있다.

## Vue와 비교

Vue의 `ref`는 `.value`가 바뀌는 mutable cell에 가깝다. 같은 `ref` 객체를 여러 함수가 잡고 있어도 `.value`를 읽는 순간 최신 값을 본다. Vue의 반응성 시스템은 이 cell을 추적하고, 값 변경이 관련 effect를 다시 실행하게 만든다.

React의 `useState`가 돌려주는 `count`는 cell이 아니다. 특정 render 호출에 주어진 숫자 값이다. `setCount`를 호출해도 이미 실행 중인 함수 안의 `count`는 바뀌지 않는다. 다음 render에서 새 `count`가 들어올 뿐이다.

이 차이는 장점도 있다. React 이벤트 핸들러는 실행 도중 state가 몰래 바뀌지 않는다. 하나의 render 안에서 값이 고정되어 있으므로 비동기 콜백이 어떤 값을 캡처했는지 코드만으로 추론하기 쉽다. 반대로 최신 값을 반드시 읽어야 하는 경우에는 updater function이나 ref를 명시적으로 선택해야 한다.

## React 내부 원리

공식 문서는 state가 컴포넌트 밖 React 내부에 저장되고, 컴포넌트 호출 시점마다 해당 render의 스냅샷으로 전달된다고 설명한다. 다음 코드는 가장 흔한 오해를 보여준다.

```tsx
import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button
      onClick={() => {
        setCount(count + 1);
        setCount(count + 1);
        setCount(count + 1);
      }}
    >
      +3: {count}
    </button>
  );
}
```

`count`가 0인 render에서 만든 handler는 세 번 모두 `setCount(1)`을 요청한다. 그래서 결과는 3이 아니라 1이다. 이전 값에 기반해 여러 번 갱신해야 하면 updater function을 써야 한다.

```tsx
setCount((n) => n + 1);
setCount((n) => n + 1);
setCount((n) => n + 1);
```

객체나 배열 state도 같은 원리다. 기존 객체를 직접 바꾸면 React에게 "새 스냅샷"이 전달되지 않는다. 새 객체를 만들어 setter에 넘겨야 다음 render가 요청되고, 이전 스냅샷과 다음 스냅샷을 안전하게 비교할 수 있다.

```tsx
setUser((user) => ({
  ...user,
  name: nextName.trim(),
}));
```

## 실무에서 터지는 문제

- **setState 직후 값을 읽고 최신이라고 믿음**: `setCount(count + 1); console.log(count)`는 이전 render의 값을 찍는다. 다음 render 결과를 보고 싶으면 UI나 effect, 또는 updater function 안에서 흐름을 설계한다.
- **비동기 콜백의 stale closure**: timeout이나 promise 콜백은 생성된 render의 값을 기억한다. 최신 값이 필요하면 ref에 보관하거나, 상태 변경 자체는 updater function으로 표현한다.
- **객체 mutation 후 화면이 안 바뀜**: `user.name = "Kim"; setUser(user)`는 같은 참조를 다시 넘기는 패턴이다. 새 객체나 새 배열을 만들어야 한다.
- **derived state를 중복 저장**: `fullName`처럼 props/state에서 계산 가능한 값을 별도 state로 두면 두 스냅샷이 어긋난다. 렌더 중 계산하거나 필요할 때 `useMemo`를 검토한다.
