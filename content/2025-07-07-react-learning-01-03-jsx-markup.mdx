## JSX로 마크업 작성하기

### JSX란?

**JSX (JavaScript XML)**는 JavaScript 파일 안에 HTML 비슷한 마크업을 쓸 수 있게 해주는 문법이야.

### 핵심 특징

1. **HTML보다 까다로워**: 태그는 무조건 닫아야 해
2. **동적 정보 표시 가능**: JavaScript 표현식을 그냥 넣을 수 있어
3. **React 컴포넌트의 기본**: 렌더링 로직과 마크업을 한 고에 놓고 관리해

### HTML을 JSX로 변환하기

HTML을 그냥 복붙하면 에러가 나. JSX는 HTML보다 규칙이 좋더 엄격하거든.

#### ❌ 작동하지 않는 코드 (HTML 그대로 복붙)

```jsx
export default function TodoList() {
  return (
    // ❌ 여러 요소를 반환할 수 없음!
    <h1>Hedy Lamarr's Todos</h1>
    <img
      src="https://i.imgur.com/yXOvdOSs.jpg"
      alt="Hedy Lamarr"
      class="photo"        // ❌ class는 JSX에서 사용 불가
    >                      // ❌ 자체 닫기 태그 필요
    <ul>
      <li>Invent new traffic lights      // ❌ 태그 닫기 필요
      <li>Rehearse a movie scene
      <li>Improve spectrum technology
    </ul>
  )
}
```

#### ✅ 제대로 고친 JSX 코드

JSX 규칙에 맞게 고친 버전이야.

```jsx
export default function TodoList() {
  return (
    <>
      <h1>Hedy Lamarr's Todos</h1>
      <img
        src="https://i.imgur.com/yXOvdOSs.jpg"
        alt="Hedy Lamarr"
        className="photo"  // ✅ className 사용
      />                   // ✅ 자체 닫기
      <ul>
        <li>Invent new traffic lights</li>  // ✅ 태그 닫기
        <li>Rehearse a movie scene</li>
        <li>Improve spectrum technology</li>
      </ul>
    </>  // ✅ Fragment로 감싸기
  );
}
```

### JSX 규칙

JSX를 쓸 때 꼭 지켜야 할 규칙들이 있어.

#### 1. **한 개의 부모 태그로 감싸야 해**

```jsx
// ❌ 여러 요소를 그냥 리턴하면 에러
return (
  <h1>Title</h1>
  <p>Content</p>
)

// ✅ Fragment(<>)로 감싸면 돼 (DOM에 추가 태그 없음)
return (
  <>
    <h1>Title</h1>
    <p>Content</p>
  </>
)

// ✅ div로 감싸도 돼 (단, DOM에 div가 추가됨)
return (
  <div>
    <h1>Title</h1>
    <p>Content</p>
  </div>
)
```

**설명:**
- React는 한 번에 한 가지만 리턴할 수 있어
- 여러 개를 리턴하려면 무언가로 감싸야 해
- `<> </>` (Fragment)는 보이지 않는 래퍼로, HTML에 태그가 추가되지 않아

#### 2. **모든 태그는 무조건 닫아야 해**

HTML에서는 `<img>`, `<br>` 같은 걸 닫지 않아도 되는데, JSX는 안 돼.

```jsx
// ❌ 잘못된 코드
<img src="...">
<br>

// ✅ 올바른 코드
<img src="..." />
<br />
```

#### 3. **속성명은 camelCase로 써야 해**

JSX는 JavaScript로 변환되기 때문에 HTML 속성명을 그대로 못 써.

```jsx
// ❌ HTML 속성명
<div class="container" onclick="handleClick()">

// ✅ JSX 속성명 (camelCase)
<div className="container" onClick={handleClick}>
```

**주요 변경사항:**

| HTML | JSX |
|------|-----|
| `class` | `className` |
| `for` | `htmlFor` |
| `onclick` | `onClick` |
| `onchange` | `onChange` |
| `tabindex` | `tabIndex` |

### HTML → JSX 변환기 활용

복잡한 HTML을 변환하기 귀찮으면 [변환기](https://transform.tools/html-to-jsx)를 써. 자동으로 바꿔줘.

### Vue와 비교

```html
<!-- Vue: HTML 문법 그대로 -->
<template>
  <div>
    <h1>{{ title }}</h1>
    <img :src="imageUrl" class="photo">
  </div>
</template>
```

```jsx
// React: JSX (더 엄격)
export default function Component() {
  return (
    <div>
      <h1>{title}</h1>
      <img src={imageUrl} className="photo" />
    </div>
  )
}
```
