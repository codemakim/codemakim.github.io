# React Router 가이드 - 중첩 라우트

> React Router 최신 Declarative Mode 문서를 기준으로 정리한 학습 자료다.
> 예제는 기존 React SPA 프로젝트에서 많이 쓰는 `react-router-dom` import를 사용한다.

---

## 왜 필요한가

1편에서는 URL 하나와 컴포넌트 하나를 연결했다. 하지만 실제 앱은 페이지마다 완전히 새 화면이 아니다.

대시보드 앱을 떠올려보자.

- 모든 화면에 상단 헤더가 있다.
- `/dashboard` 아래 화면에는 사이드바가 있다.
- `/dashboard/projects`와 `/dashboard/settings`는 사이드바를 공유한다.
- 바뀌는 부분은 오른쪽 본문뿐이다.

중첩 라우트는 이 구조를 라우트 트리로 표현한다. 상위 라우트는 공통 레이아웃을 렌더링하고, 하위 라우트는 그 안의 특정 위치에 들어간다.

그 특정 위치가 `<Outlet />`이다.

---

## 원리: 부모 화면 안에 자식 화면이 들어간다

React Router에서 라우트를 중첩하면 자식 경로는 부모 경로를 이어받는다.

```tsx
<Route path="/dashboard" element={<DashboardLayout />}>
  <Route index element={<DashboardHomePage />} />
  <Route path="projects" element={<ProjectsPage />} />
</Route>
```

이 설정은 두 주소를 만든다.

- `/dashboard`
- `/dashboard/projects`

주소가 `/dashboard/projects`일 때 렌더링 흐름은 다음과 같다.

```
DashboardLayout 렌더링
  ↓
DashboardLayout 안의 <Outlet /> 위치 탐색
  ↓
ProjectsPage를 그 자리에 렌더링
```

부모 컴포넌트에 `<Outlet />`이 없으면 자식 라우트가 매칭되어도 화면에 나타날 자리가 없다.

---

## 최소 예제

먼저 가장 작은 중첩 라우트를 만든다.

```tsx
// App.tsx
import { Link, Outlet, Route, Routes } from "react-router-dom";

function DashboardLayout() {
  return (
    <section>
      <h1>Dashboard</h1>
      <nav>
        <Link to="/dashboard">Overview</Link>
        {" | "}
        <Link to="/dashboard/settings">Settings</Link>
      </nav>

      <Outlet />
    </section>
  );
}

function DashboardHomePage() {
  return <p>대시보드 홈이다.</p>;
}

function SettingsPage() {
  return <p>설정 화면이다.</p>;
}

export default function App() {
  return (
    <Routes>
      <Route path="/dashboard" element={<DashboardLayout />}>
        <Route index element={<DashboardHomePage />} />
        <Route path="settings" element={<SettingsPage />} />
      </Route>
    </Routes>
  );
}
```

핵심은 세 가지다.

- `DashboardLayout`은 공통 UI를 담당한다.
- `<Outlet />`은 자식 화면이 들어갈 자리다.
- `index` 라우트는 `/dashboard`에 정확히 들어왔을 때 보여줄 기본 자식 화면이다.

## Layout Route

상위 라우트가 URL 조각을 추가하지 않고 레이아웃만 제공할 수도 있다. 이런 라우트를 Layout Route라고 부른다.

```tsx
<Routes>
  <Route element={<RootLayout />}>
    <Route index element={<HomePage />} />
    <Route path="login" element={<LoginPage />} />
    <Route path="dashboard" element={<DashboardLayout />}>
      <Route index element={<DashboardHomePage />} />
      <Route path="settings" element={<SettingsPage />} />
    </Route>
  </Route>
</Routes>
```

위 예제에서 `RootLayout` 라우트는 `path`가 없다. 그래서 URL에는 아무 조각도 더하지 않는다. 대신 모든 자식 라우트에 공통 헤더나 페이지 폭 같은 레이아웃을 제공한다.

```tsx
// layouts/RootLayout.tsx
import { Link, Outlet } from "react-router-dom";

export default function RootLayout() {
  return (
    <div>
      <header>
        <Link to="/">Home</Link>
        {" | "}
        <Link to="/dashboard">Dashboard</Link>
        {" | "}
        <Link to="/login">Login</Link>
      </header>

      <main>
        <Outlet />
      </main>
    </div>
  );
}
```

URL을 만들고 싶으면 `path`를 둔다. 레이아웃만 만들고 싶으면 `path` 없이 `element`와 자식 라우트만 둔다.

---

## 실무 예제: 대시보드 레이아웃

조금 더 실제에 가까운 예제를 보자.

```tsx
// App.tsx
import { Route, Routes } from "react-router-dom";
import RootLayout from "./layouts/RootLayout";
import DashboardLayout from "./layouts/DashboardLayout";
import HomePage from "./pages/HomePage";
import LoginPage from "./pages/LoginPage";
import NotFoundPage from "./pages/NotFoundPage";
import DashboardHomePage from "./pages/dashboard/DashboardHomePage";
import ProjectsPage from "./pages/dashboard/ProjectsPage";
import SettingsPage from "./pages/dashboard/SettingsPage";

export default function App() {
  return (
    <Routes>
      <Route element={<RootLayout />}>
        <Route index element={<HomePage />} />
        <Route path="login" element={<LoginPage />} />

        <Route path="dashboard" element={<DashboardLayout />}>
          <Route index element={<DashboardHomePage />} />
          <Route path="projects" element={<ProjectsPage />} />
          <Route path="settings" element={<SettingsPage />} />
        </Route>

        <Route path="*" element={<NotFoundPage />} />
      </Route>
    </Routes>
  );
}
```

`DashboardLayout`은 대시보드 영역의 공통 UI만 책임진다.

```tsx
// layouts/DashboardLayout.tsx
import { NavLink, Outlet } from "react-router-dom";

export default function DashboardLayout() {
  return (
    <div style={{ display: "grid", gridTemplateColumns: "220px 1fr", gap: 16 }}>
      <aside>
        <h2>Dashboard</h2>
        <nav>
          <NavLink to="" end>
            Overview
          </NavLink>
          <br />
          <NavLink to="projects">Projects</NavLink>
          <br />
          <NavLink to="settings">Settings</NavLink>
        </nav>
      </aside>

      <section>
        <Outlet />
      </section>
    </div>
  );
}
```

여기서 `to="projects"`는 상대 링크다. 현재 라우트가 `/dashboard`이므로 최종 주소는 `/dashboard/projects`가 된다. 상위 경로가 바뀌어도 하위 메뉴 링크를 덜 고치게 된다.

## Index Route 기준

`index` 라우트는 “부모 주소에 정확히 들어왔을 때 보여줄 기본 자식 화면”이다.

```tsx
<Route path="dashboard" element={<DashboardLayout />}>
  <Route index element={<DashboardHomePage />} />
  <Route path="settings" element={<SettingsPage />} />
</Route>
```

- `/dashboard` → `DashboardHomePage`
- `/dashboard/settings` → `SettingsPage`

`index` 라우트는 자식을 가질 수 없다. 기본 화면이면서 동시에 더 깊은 레이아웃이 필요하다면 `index`가 아니라 별도 Layout Route를 써야 한다.

---

## 함정과 디버깅

### 1. `Outlet`을 빼먹는다

부모 레이아웃은 보이는데 자식 화면이 보이지 않으면 `<Outlet />`부터 확인한다. 자식 라우트는 자동으로 부모 컴포넌트 아래 아무 데나 붙지 않는다.

### 2. 자식 `path`에 `/`를 붙인다

```tsx
// 피해야 하는 예
<Route path="dashboard" element={<DashboardLayout />}>
  <Route path="/settings" element={<SettingsPage />} />
</Route>
```

자식 라우트에서 `/settings`처럼 시작하면 절대 경로처럼 읽힌다. 중첩 구조를 표현하려면 보통 `settings`처럼 상대 경로를 쓴다.

### 3. 루트 메뉴가 항상 활성화된다

`NavLink`는 현재 URL과 링크의 매칭 여부를 계산한다. 루트(`/`) 링크는 모든 주소의 시작처럼 보일 수 있으므로 `end`를 붙여 정확히 `/`일 때만 활성화한다.

```tsx
<NavLink to="/" end>
  Home
</NavLink>
```

### 4. 레이아웃 컴포넌트가 데이터를 너무 많이 안다

레이아웃은 공통 UI와 공통 컨텍스트만 책임지는 편이 좋다. 각 페이지 고유 데이터는 자식 페이지에서 다루면 라우트 구조가 오래 버틴다.

---

## Vue Router와 비교

Vue Router의 `children`과 React Router의 중첩 라우트는 같은 문제를 푼다.

| 관심사             | Vue Router        | React Router             |
| ------------------ | ----------------- | ------------------------ |
| 중첩 정의          | `children` 배열   | `<Route>` 안의 `<Route>` |
| 자식 표시 위치     | `<router-view />` | `<Outlet />`             |
| 기본 자식          | `path: ""`        | `index`                  |
| 경로 없는 레이아웃 | 부모 route 객체   | `path` 없는 `<Route>`    |
| 활성 링크          | `<router-link>`   | `<NavLink>`              |

Vue에서는 라우트 객체를 먼저 떠올리는 경우가 많고, React Router에서는 컴포넌트 트리 안에서 “부모가 자식을 어디에 렌더링하는가”를 더 직접적으로 본다.

---

## 체크리스트

- 중첩 라우트가 공통 레이아웃 재사용 문제를 해결한다는 것을 설명할 수 있다.
- `<Outlet />`이 자식 라우트의 렌더링 위치라는 것을 이해한다.
- `index` 라우트와 `path`가 있는 자식 라우트를 구분할 수 있다.
- `path` 없는 Layout Route가 URL 조각을 추가하지 않는다는 것을 안다.
- 자식 경로와 상대 링크를 사용할 수 있다.
- `NavLink`의 `end`가 필요한 상황을 설명할 수 있다.
