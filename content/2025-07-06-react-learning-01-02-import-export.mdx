## 컴포넌트 Import/Export

### 문제 상황

한 파일에 컴포넌트를 너무 많이 넣으면 파일이 길어지고 찾기 힘들어져.

### 해결책: 파일 분리

**파일 구조:**

```
src/
  ├── Gallery.js      # Gallery 컴포넌트
  └── Profile.js      # Profile 컴포넌트
```

### Export/Import 예제

Profile 컴포넌트를 별도 파일로 빼내고, Gallery에서 불러와서 쓰는 방법을 보자.

**1단계: Profile.js 파일 생성**

```jsx
// Profile.js
export default function Profile() {
  return (
    <img
      src="https://i.imgur.com/MK3eW3As.jpg"
      alt="Katherine Johnson"
    />
  );
}
```

- `export default`를 앞에 붙이면 다른 파일에서도 이 컴포넌트를 가져다 쓸 수 있어.
- 파일 하나에 메인 컴포넌트가 하나만 있을 때 default export를 써.

**2단계: Gallery.js에서 불러오기**

```jsx
// Gallery.js
import Profile from './Profile.js';

export default function Gallery() {
  return (
    <section>
      <h1>Amazing scientists</h1>
      <Profile />
      <Profile />
      <Profile />
    </section>
  );
}
```

- `import Profile from './Profile.js'`: Profile 컴포넌트를 가져와.
- `'./Profile.js'`: 현재 폴더(`.`)에 있는 Profile.js 파일이라는 뜻이야.
- 이제 `<Profile />`를 쓸 수 있어.

### Export 방식

React에서는 두 가지 Export 방식이 있어.

| 방식 | Export | Import | 설명 |
|------|--------|--------|------|
| **Default Export** | `export default function Button() {}` | `import Button from './Button.js'` | 파일당 하나의 주요 컴포넌트 |
| **Named Export** | `export function Button() {}` | `import { Button } from './Button.js'` | 파일당 여러 컴포넌트 |

**차이점:**

1. **Default Export**:
   - 한 파일에 하나만 가능
   - Import할 때 중괄호 `{}` 없이 써
   - Import할 때 이름을 마음대로 바꿀 수 있어: `import MyButton from './Button.js'`

2. **Named Export**:
   - 한 파일에 여러 개 넣을 수 있어
   - Import할 때 반드시 중괄호 `{}` 써야 해
   - 정확한 이름으로 Import 해야 돼

**언제 뭘 써야 할까?**

- 파일당 컴포넌트 하나 → **Default Export** 써 (대부분의 경우)
- 파일당 컴포넌트 여러 개 → **Named Export** 써

### Vue와 비교

```html
<!-- Vue: Profile.vue -->
<template>
  <img src="..." alt="Katherine Johnson" />
</template>

<script setup>
// 자동으로 export됨
</script>
```

```jsx
// React: Profile.js
export default function Profile() {
  return <img src="..." alt="Katherine Johnson" />
}
```
